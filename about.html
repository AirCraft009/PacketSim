Simulator Limitations

This simulator is designed for educational purposes and provides a simplified representation of network components and their interactions.

  • Each subnet requires its own router as a gateway.
  • No Layer 2 behavior is simulated.
  • Switches are intentionally omitted.
  • Packet loss is artificially generated and does not reflect real-world behavior.
  • Advanced routing protocols, security mechanisms, and performance optimizations are not implemented.

Network Components

Router
Global & local routing

Central component of the simulator.
Uses SPF to compute shortest paths between networks.
Always acts as the subnet gateway.

SPF / Dijkstra Algorithm
PC
Packet origin

Manual packet sending is only possible from PCs.
Other hosts could originate traffic, but this is avoided to reduce simulator complexity.

Server
Packet response

Servers wait for incoming packets and respond with an echo unless additional behavior is implemented.
