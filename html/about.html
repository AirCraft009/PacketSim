How the Simulator Works

The simulator follows a simple workflow designed to make network behavior observable rather than realistic.

1. Build

Place routers and hosts to define subnets. Each network must have exactly one router acting as its gateway.

2. Send

Create packets manually from PCs. This keeps traffic generation explicit and easy to follow.

3. Route

Routers calculate paths automatically using shortest-path logic and forward packets accordingly.

4. Observe

Watch packets traverse the network step by step, making routing decisions visible.

Simulator Limitations

This simulator is designed for educational purposes and provides a simplified representation of network components and their interactions.

  • Each subnet requires its own router as a gateway.
  • No Layer 2 behavior is simulated.
  • Switches are intentionally omitted.
  • Advanced routing protocols, security mechanisms, and performance optimizations are not implemented.

Our Mission

NetSim exists to teach fundamental networking concepts without the overhead of real hardware, operating systems, or vendor-specific tooling.

The simulator focuses on core ideas such as packet forwarding, routing decisions, subnet separation, and end-to-end communication. By intentionally omitting lower-level details and advanced protocols, the system keeps cause-and-effect relationships visible and predictable.

This makes NetSim suitable for beginners who want to understand how networks behave conceptually before dealing with real-world complexity.
